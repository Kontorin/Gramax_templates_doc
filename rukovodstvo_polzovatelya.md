# 1. Введение

### 1.1. Область применения
Данное руководство предназначено для пользователей внутреннего финансового сервиса компании, обеспечивающего автоматизированный расчёт заработной платы, премий, удержаний и формирование отчетных документов. Руководство применяется сотрудниками бухгалтерии, отдела кадров и руководителями подразделений.

### 1.2. Краткое описание возможностей
Внутренний финансовый сервис предоставляет следующие возможности:  
- автоматический расчёт заработной платы сотрудников;  
- начисление премий по установленным правилам;  
- формирование отчетов по ФОТ, начислениям и выплатам;  
- загрузка и выгрузка данных сотрудников;  
- просмотр индивидуальных начислений;  
- настройка коэффициентов, формул и премиальных схем;  
- работа с историей начислений и логами изменений;  
- интеграция с кадровой системой через API.

### 1.3. Уровень подготовки пользователя
Для работы с системой требуется:  
- базовое знание ПК;  
- умение работать с веб-интерфейсом;  
- понимание внутренних регламентов начисления зарплаты;  
- для администраторов – базовые навыки управления ролями и доступами.

### 1.4. Перечень эксплуатационной документации
Пользователь должен ознакомиться со следующими документами:  
- Спецификация требований;  
- Техническое задание;  
- Руководство администратора;  
- Внутренние регламенты расчёта зарплат и премий;  
- Инструкция по работе с корпоративной системой авторизации.

# 2. Назначение и условия применения

### 2.1. Виды деятельности и функции
Система предназначена для автоматизации следующих процессов:  
- учёт сотрудников и их кадровых данных;  
- расчёт окладов, надбавок, удержаний;  
- начисление премий по различным схемам;  
- управление периодами расчёта;  
- формирование отчётности и экспорт данных;  
- ведение журналов операций и истории изменений.

### 2.2. Условия применения
Для корректной работы системы необходимо:  
- использование ПК или ноутбука с поддержкой современных браузеров (Chrome, Firefox, Edge, Safari);  
- подключение к корпоративной сети;  
- наличие учетной записи в системе авторизации компании;  
- наличие роли «бухгалтер», «кадровый специалист» или «администратор»;  
- доступ к корпоративной базе данных (для администраторов);  
- предварительная загрузка данных о сотрудниках и их текущих кадровых параметрах.

---

# 3. Подготовка к работе

### 3.1. Состав дистрибутивного носителя
Веб-сервис разворачивается на сервере компании и включает:  
- веб-интерфейс пользователя;  
- модуль расчёта;  
- модуль отчетности;  
- корпоративную СУБД;  
- API-интерфейсы для интеграции;  
- документацию в формате Markdown и Word;  
- диаграммы архитектуры в PlantUML.

### 3.2. Порядок загрузки данных и программ
Пользователь получает доступ к системе через корпоративный портал:  
1. Перейти по ссылке на внутренний финансовый сервис.  
2. Авторизоваться через корпоративный аккаунт.  
3. Дождаться загрузки пользовательского интерфейса.  
4. При необходимости загрузить данные сотрудников через модуль "Импорт".  
5. Проверить корректность данных в разделе «Сотрудники».  

### 3.3. Порядок проверки работоспособности
Для проверки корректной работы системы необходимо:  
1. Убедиться, что отображаются списки сотрудников.  
2. Выполнить тестовый расчёт периода на небольшой выборке.  
3. Проверить формирование отчёта.  
4. Убедиться, что расчёты можно экспортировать.  
5. Проверить корректность ролей и доступов.

# 4. Описание операций

## 4.1. Выполняемые функции и задачи
Система позволяет выполнять следующие задачи:  
- запуск расчёта зарплатного периода;  
- автоматическое начисление премий;  
- редактирование индивидуальных начислений;  
- формирование Excel и PDF отчётов;  
- просмотр истории изменений и логов операций;  
- экспорт расчётных ведомостей;  
- управление данными сотрудников;  
- управление периодами расчёта.

## 4.2. Описание операций технологического процесса обработки данных

### Операция 1. Запуск расчёта зарплаты
- **Условия:** данные сотрудников актуальны и проверены.  
- **Подготовительные действия:** выбрать расчётный период.  
- **Основные действия:**  
  1. Перейти в раздел «Расчёт зарплаты».  
  2. Нажать «Запустить расчёт».  
  3. Дождаться завершения обработки.  
- **Заключительные действия:** просмотреть результаты, сохранить или экспортировать отчёт.  
- **Ресурсы:** вычислительные мощности сервера.

### Операция 2. Начисление премий
- **Условия:** премиальные коэффициенты настроены.  
- **Подготовка:** проверить параметры премирования.  
- **Основные действия:**  
  1. Открыть вкладку «Премии».  
  2. Выбрать схему премирования.  
  3. Нажать «Начислить».  
- **Заключительные действия:** сформировать отчёт.  

### Операция 3. Импорт данных сотрудников
- **Условия:** файл соответствует формату шаблона.  
- **Подготовка:** подготовить таблицу в формате CSV или Excel.  
- **Основные действия:**  
  1. Перейти в «Импорт данных».  
  2. Загрузить файл.  
  3. Проверить предварительный просмотр.  
- **Заключение:** подтвердить загрузку.  

### Операция 4. Экспорт отчётов
- **Условия:** завершён расчёт периода.  
- **Основные действия:**  
  1. Перейти в «Отчёты».  
  2. Выбрать формат (PDF, Excel).  
  3. Скачать готовый файл.

# 5. Аварийные ситуации

В случае возникновения нестандартных ситуаций пользователь должен:

### 1. Отказ серверного оборудования или недоступность системы
- Уведомить ИТ-отдел.  
- Не предпринимать попыток повторного входа до восстановления связи.

### 2. Ошибки данных или некорректные начисления
- Проверить исходные данные сотрудников.  
- Перезапустить расчёт периода.  
- При сохранении ошибки — обратиться к администратору системы.

### 3. Ошибка при импорте данных
- Проверить формат и структуру файла.  
- Пересохранить файл в нужном формате.  
- Повторить импорт.

### 4. Подозрение на несанкционированный доступ
- Немедленно сообщить службе безопасности.  
- Заблокировать доступ к своей учётной записи через портал.

# 6. Рекомендации по освоению

- Начать работу с изучения разделов интерфейса.  
- Выполнить тестовый расчёт периода на небольшом количестве данных.  
- Ознакомиться с примером расчётной ведомости.  
- Использовать встроенные подсказки в интерфейсе.  
- Обратиться к администратору в случае возникновения вопросов.  
- Регулярно проверять обновления документации в корпоративном репозитории.

Контрольный пример:  
1. Загрузить тестовый список сотрудников (5–10 человек).  
2. Запустить расчёт периода.  
3. Начислить премии базовой схемой.  
4. Сформировать и скачать отчёт в Excel.  
5. Сверить значения с методическими материалами.
