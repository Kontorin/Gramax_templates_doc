# Руководство администратора  
Автоматизированная система расчёта заработной платы и премий сотрудников  
(АСРЗП)

Документ разработан в соответствии с ГОСТ 19.505–79, ГОСТ 19.506–79 и требованиями к эксплуатационной документации серии 19.

---

# 1. Область применения

Настоящее Руководство администратора предназначено для системных администраторов, специалистов ИТ-подразделения и лиц, ответственных за установку, техническое сопровождение и администрирование автоматизированной системы расчёта заработной платы и премий сотрудников (АСРЗП).

Документ описывает:
- порядок установки и конфигурирования системы;
- управление пользователями, ролями и правами доступа;
- порядок обновления программного обеспечения;
- действия администратора в аварийных ситуациях;
- процедуры мониторинга работоспособности и обеспечения безопасности.

---

# 2. Назначение системы и условия применения

## 2.1. Назначение системы
АСРЗП обеспечивает:
- ведение данных сотрудников компании;
- автоматический расчёт заработной платы и премий;
- формирование ведомостей выплат;
- обеспечение контроля доступа к финансовым данным.

## 2.2. Условия применения
Для корректной работы системы необходимо следующее:

### Технические требования сервера
- ОС: Linux Ubuntu Server 20.04+ или Windows Server 2019+  
- CPU ≥ 4 ядра  
- RAM ≥ 16 GB  
- SSD ≥ 100 GB  
- СУБД PostgreSQL 13+  
- Python 3.10+  

### Сетевые условия
- Доступ к внутренней сети компании.
- Использование защищённого канала (HTTPS).

### Требования к персоналу
Администратор должен:
- иметь навыки работы с Linux/Windows Server;
- знать основы SQL и управления PostgreSQL;
- разбираться в сетевой инфраструктуре.

# 5. Управление пользователями и доступом

## 5.1. Роли пользователей

| Роль | Описание | Права |
|------|----------|--------|
| **Администратор** | техническое сопровождение | доступ ко всей системе |
| **Финансист** | расчёт зарплат и премий | изменение данных сотрудников, расчёты |
| **Руководитель подразделения** | просмотр подчинённых | просмотр ведомостей |
| **Наблюдатель** | контролирующая роль | чтение отчётов |

## 5.2. Добавление нового пользователя
1. Перейдите в административную панель: `/admin`.  
2. Откройте раздел **Пользователи**.  
3. Нажмите **Создать пользователя**.  
4. Заполните поля:
   - логин  
   - ФИО  
   - подразделение  
   - роль  
5. Установите временный пароль.  
6. Сохраните запись.

## 5.3. Управление правами доступа
Каждая роль имеет набор предопределённых разрешений.  
Администратор может создавать кастомные роли через раздел **Роли**.

# 7. Аварийные ситуации и их обработка

| Ситуация | Причина | Действия администратора |
|----------|---------|--------------------------|
| Система не запускается | Ошибка конфигурации | Проверить логи `journalctl -u asrzp` |
| Нет связи с БД | Сбой PostgreSQL | Перезапустить `systemctl restart postgresql` |
| Некорректные расчёты | Ошибка данных | Проверить справочники, историю изменений |
| Пользователь забыл пароль | Сброс пароля | В админ-панели нажать «Сбросить пароль» |
| Медленная работа системы | Нагрузка на сервер | Проверить ресурсы: `top`, `iotop` |

# 8. Обеспечение безопасности

- Использование HTTPS обязательно.  
- Пароли хранятся только в зашифрованном виде (bcrypt/argon2).  
- Доступ к серверу ограничен по IP.  
- Все действия пользователя записываются в журнал аудита.  
- Раз в 90 дней проводится проверка актуальности прав.

# 9. Рекомендации по эксплуатации

- Проводить регулярные обновления системы.  
- Еженедельно проверять целостность базы данных.  
- Контролировать свободное место на сервере.  
- Проводить обучение пользователей согласно регламенту.  
- Хранить резервные копии не менее 90 дней.
