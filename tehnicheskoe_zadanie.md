## 2. Цели и назначение автоматизированной системы

### 2.1. Цели создания АС
- Автоматизация расчёта заработной платы сотрудников.
- Исключение ошибок ручного расчёта.
- Автоматизация начисления премий.
- Ускорение финансовых процедур.
- Централизация данных сотрудников и расчётов.

**Критерии достижения целей:**
- 100% данных сотрудников ведутся в системе.
- Полностью автоматизирован расчёт окладов, налогов и премий.
- Время расчёта одного сотрудника ≤ 1 сек.
- Система проходит тесты на корректность всех формул.

### 2.2. Назначение АС
АС предназначена для автоматизации процессов расчёта окладов, налогов, удержаний, премий и формирования итоговых ведомостей.

## 3. Характеристика объекта автоматизации

1. **Объект:** процесс финансового расчёта зарплат и премий сотрудников.

2. **Условия эксплуатации:**

## 4. Требования к автоматизированной системе

### 4.1. Требования к структуре АС

АС включает следующие подсистемы:
- Ведение данных сотрудников.
- Расчёт заработной платы.
- Расчёт премий.
- Администрирование и управление доступом.
- Формирование отчётности.

**Технологическое взаимодействие:** REST API + PostgreSQL.

**Интеграции:**
- Импорт кадровых данных (CSV/JSON).
- Экспорт ведомостей в бухгалтерский комплекс.

**Режимы работы:**
- Интерактивный пользовательский режим.
- Регламентные массовые расчёты.

### 4.2. Требования к функциям системы

#### Основные функции
1. Создание, редактирование и удаление данных сотрудников.
2. Автоматический расчёт окладной части.
3. Расчёт налогов: НДФЛ, страховые взносы.
4. Поддержка премий: фиксированных, процентных, коэффициентных.
5. Формирование ведомостей начисления.
6. Экспорт отчётов в PDF/XLSX.
7. Журналирование всех изменений.

#### Функциональные требования
- Время формирования ведомости (до 500 сотрудников) ≤ 5 секунд.
- Все расчёты выполняются с точностью до двух знаков.

#### Отказы
- Ошибки должны логироваться.
- Потеря данных при сбое недопустима.
- Автоматическое восстановление транзакций.

### 4.3. Требования к видам обеспечения

#### 4.3.1. Информационное обеспечение
- Таблицы: сотрудники, должности, оклады, премии, налоги.
- Ежедневное резервное копирование.
- Справочники: должности, коэффициенты премирования, ставки налогов.

#### 4.3.2. Лингвистическое обеспечение
- Интерфейс на русском языке.
- Возможность добавления английской локализации.

#### 4.3.3. Программное обеспечение
- Backend: Python 3.10 / Django.
- Frontend: React или Qt (по внутреннему стандарту).
- Отчётность: ReportLab, XLSXWriter.
- СУБД: PostgreSQL 13+.

#### 4.3.4. Техническое обеспечение
**Рабочие места:**
- CPU ≥ 2.5 GHz  
- RAM ≥ 8 GB  
- HDD ≥ 10 GB  

**Сервер:**
- CPU ≥ 4 ядра  
- RAM ≥ 16 GB  
- SSD ≥ 100 GB  

#### 4.3.5. Метрологическое обеспечение
- Точность расчётов: 0.01.
- Закреплённые формулы – по внутренним регламентам.

#### 4.3.6. Организационное обеспечение
- Подразделения: ИТ и Финансовый отдел.
- Обучение пользователей.
- Регламент действий при сбоях.

#### 4.3.7. Методическое обеспечение
- Используемые стандарты: ГОСТ 19.101–77, ГОСТ 34.602–2020, ISO/IEC 26514.
